# ⚙️ 用戶端本機配置

## 配置讀取

配置檔預設路徑位於系統使用者目錄下的`.mdc/mdc.ini`。

如果程式所在目錄中也存在名為`mdc.ini`的配置檔，程式會優先讀取該檔案。這樣可以讓不同位置的程式讀取獨立配置檔。

## 目錄配置

當`刮削目錄`之一與`輸出目錄`相同時，可實現在刮削目錄下直接刮削與整理。此模式下只能配置 1 個刮削目錄。

### 刮削目錄

預設為程式所在資料夾，可自行新增多個資料夾。

### 輸出目錄

預設為程式運行目錄下新建的`output`資料夾。

## 其他配置

### 名稱提取模式

* 基於影片檔案命名，對影片進行名稱提取測試

每次運行前，請務必勾選後執行一次檢查名稱提取。

如果提取的名稱與影片名稱不一致，請按照[命名規則](/zh-Hant/naming.md)手動修改影片檔案名稱後，再執行一次進行檢查。

直至要刮削的影片名稱提取都正確，才能取消勾選名稱提取模式後運行。

### 偵錯模式

* 務必保持常閉，出現 BUG 時方可開啟，否則會增加程式崩潰概率

請在出現 BUG 過程中點擊`停止`，勾選`偵錯模式`後再次點擊`執行`，保存輸出日誌和相關截圖後按照[報告 BUG](/zh-Hant/bug_report.md)發送。

### 檢查更新

* 建議常開以獲取最新更新提醒

## 自訂網址刮削

* 點擊`+`新增影片文件
* 網址填寫為影片資訊的具體網址，如 [https://www.themoviedb.org/movie/13189-a-christmas-carol](https://www.themoviedb.org/movie/13189-a-christmas-carol)
* 支援的影片資訊網址，為雲端配置中的通用網站與命名規則中影片類型的網站
* 運行過程中，若處理有自訂網址的影片，則會使用自訂網址

## 演員頭像

::: tip
僅支援 Emby、Jellyfin。
:::

* 輸入伺服器地址
* 輸入 API 金鑰，或點擊進入`金鑰創建頁面`，新建 API 金鑰後複製到輸入框
* 點擊`保存並測試`
* 點擊`開始補全頭像`

## 自訂字段對應表

* 程式首次運行後，會在系統使用者目錄中的`.mdc`目錄下產生`mapping_tables`目錄，包含兩個 XML 文件，可配合雲端配置中的`字段轉換設定`使用
* 演員映射表：`mapping_actor.xml`
* 其他字段映射表：`mapping_info.xml`
* 簡轉繁對應`zh_tw`，繁轉簡對應`zh_cn`，英文`en`只支援自訂映射表中的內容
